# Cosmic Talent Bridge (CTB)

## Master Development Rules

---

## Project Overview

This is an existing Next.js project for **Cosmic Talent Bridge (CTB)**.

The objective is to improve the existing project into a professional, enterprise-grade website without changing its core architecture.

This is a production website for a real company, not a demo or template.

---

# Primary Objective

Transform the current website into a modern, responsive, accessible and SEO-friendly platform while preserving the existing project structure.

The website should represent CTB as a company providing:

* Recruitment
* Global Staffing
* Professional Training
* Professional Verification
* HR Advisory
* Business Consulting
* Website Development
* AI Solutions
* SaaS Solutions
* Career Development

---

# Mandatory Rules

These rules apply to every future task.

## Never change

* Project architecture
* Folder structure
* Routing
* JSON data structure
* Existing reusable components
* Existing branding
* Existing CTB logo

unless absolutely necessary to fix a real technical issue.

If a change is required, explain why before making it.

---

# Existing Homepage

Preserve and improve the existing homepage.

Current structure:

Header

↓

Hero Carousel

↓

Hiring & Projection

↓

Future sections

Do not redesign completed sections unless requested.

Improve them while keeping the same overall appearance.

---

# Design Standards

The website should look like an international corporate company.

Design principles:

* Clean
* Professional
* Premium
* Modern
* Minimal
* Responsive

Avoid unnecessary animations or decorative effects.

---

# Colour Scheme

Primary

CTB Blue

Secondary

White

Text

Black

Supporting

Light Gray

Maintain a consistent colour palette across all pages.

---

# Typography

Use clean, modern typography.

Maintain consistent spacing, margins and alignment.

Avoid inconsistent font sizes.

---

# Responsive Design

Every page must work correctly on:

* Desktop
* Laptop
* Tablet
* Mobile

No horizontal scrolling.

Touch-friendly controls on mobile.

---

# Accessibility

Use semantic HTML.

Include:

* aria-labels
* alt text
* keyboard accessibility
* visible focus states
* sufficient colour contrast

---

# Performance

Improve performance without changing functionality.

Optimise:

* Images
* Lazy loading
* Rendering
* Bundle size

Avoid unnecessary JavaScript.

---

# SEO

Every page should be prepared for:

* Unique title
* Meta description
* Open Graph
* Twitter Card
* Structured Data
* Sitemap
* Robots.txt

Use semantic headings.

Do not create fake SEO claims or keyword stuffing.

---

# Code Quality

Prefer:

* Small reusable components
* Clean code
* Readable naming
* Existing utilities

Remove dead code only after confirming it is unused.

---

# Future Growth

Prepare the project for future expansion.

Possible future modules:

* Employer Portal
* Candidate Portal
* Recruiter Portal
* Training Portal
* Verification Portal
* Admin Dashboard

Do not implement these modules yet.

Only keep the architecture ready.

---

# Workflow

For every future task:

1. Understand the requirement.
2. Preserve existing architecture.
3. Make only the requested changes.
4. Keep the website stable.
5. Build successfully.
6. Test on desktop and mobile.
7. Avoid introducing new issues.

---

# Final Rule

When there are multiple possible implementations:

Choose the solution that is:

* Easier to maintain
* More scalable
* More reusable
* More accessible
* Better for SEO
* Better for long-term growth

while preserving the existing CTB project.
